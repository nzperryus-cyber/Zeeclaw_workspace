# Memory — 2026-03-31

## LCM Token Issue Discovered

Nathan discovered the root cause of massive token usage (20M+ on 3/19, 26.9M on 3/29, 20.2M on 3/30):
- LCM (lossless-claw) was installed but NOT activated as the context engine
- `plugins.slots.contextEngine` was not set
- Without LCM, OpenClaw used built-in sliding window and was sending FULL conversation history on every API call
- Fix applied: Added `"slots": {"contextEngine": "lossless-claw"}` to openclaw.json, gateway restarted
- Now LCM will: intercept messages, store in SQLite (lcm.db), summarize at 75% context usage
- The 20M tokens already spent; fix prevents future blowup
